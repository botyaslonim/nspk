Приложение выполнено в виде SPA, на Angular 1.5.6 + Bootstrap 3.3.6 + jQuery 2.2.4
Папка /root/apache-tomcat-7.0.70/webapps/x3x/resources

В начале работы с системой пользователю предлагается зайти под логином и паролем. 
Страница логина расположена по отдельному адресу и не относится собственно к приложению: /root/apache-tomcat-7.0.70/webapps/x3x/WEB-INF/view/login.jsp
Редирект на неё (при первом заходе, а также при выходе из приложения) производится сервером, со стороны front-end никаких специальных действий не требуется.


Структура папок приложения следующая:
core
css
img
js
tables
Кроме того, в корне приложения лежит файл index.html


Папка core
В этой папке лежит файл core.module.js, в который вынесены общие для всего приложения функции (сервисы)
Краткое описание:
а) Сервис popup - отвечает за отображение и работу всплывающих окон
б) Сервис header - отвечает за отбражение и работу блока выбора дат
в) Сервис sort - отвечает за сортировку данных
г) Сервис parseDoc - отвечает за разбор и выведение содержимого документа, а также построение стилизованного XML-дерева
д) Сервис ids - отвечает за маршрутизацию на вкладке "Контроль итогов обработки ED230"
е) Сервис requests - отвечает за адреса запросов на сервер


Папка css
В этой папке лежат следущие файлы css-стилей:
angular-datepicker.css - стили для каленадрей выбора дат
bootstrap.min.css, bootstrap-theme.min.css - стили Bootstrap
style.css - кастомные стили приложение, а также переопределение некоторых стилей Bootstrap


Папка img
В этой папке лежат пиктограммы инструмента сортировок колонок таблиц и иконка лоадера


Папка js
В этой папке лежат файлы Angular, Bootstrap, jQuery, а также файлы приложения:
а) app.config.js - файл маршрутизации приложения
б) app.module.js - основной файл всего приложения, которое задано в файле index.html как <html ng-app="nspk" ng-controller="main">.
Функции для работы с содержимым основного шаблона приложения index.html (меню, попап, меню пользователя), а также дефолтные значения глобальных переменных


Папка tables
В этой папке находятся файлы для отображения содержимого вкладок. Для каждой из вкладок первого уровня своя папка: table1, table2 и так далее. 7-я папка содержит внутренние папки, в соответствии со схемой навигации по содержимому на 7 вкладке.
Каждая страница приложения имеет свой типовой набор файлов. На примере первой страницы:
а) table1.component.js - файл скриптов для данной страницы
б) table1.module.js - файл подключения модуля страницы с перечислением требуемых зависимостей
в) table1.template.html - файл шаблона отображения страницы


Файл index.html
Является основным шаблоном приложения, содержит вёрстку основных блоков, а также контейнер для отображения содержимого <div ng-view></div>.
В начале файла подключаются все потребующиеся при работе приложения скрипты, а также библиотеки и файлы стилей.


Некоторые особенности
а) На 1-й и 5-й странице содержимое страниц обновляется раз в 5 секунд. Это сделано для того, чтобы когда оператор кладёт файл в папку на сервере, он начинал отображаться на странице без её перезагрузки.
б) В любом запросе на сервер, если содержимое возвращается в виде HTML-страницы, делаем вывод о том, что окончилось время сессии. Страница принудительно перезагружается, сервер редиректит на страницу логина.
в) Если содержимое страницы грузится долго (сервер опаздывает с ответом), через 3 секунды показывается предупреждение "Нужно немного подождать" и анимированная иконка лоадера, через 10 секунд пользователь видит сообщение "К сожалению, нет связи с сервером" и кнопку, по нажатию на которую происходит перезагрузка страницы.
г) Кнопка "Сохранить" попапа позволяет сохранить содержимое документа в файл. По умолчанию это XML-формат (переменная $scope.$root.popupSaveButtonFormat).
